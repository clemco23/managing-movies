Une requête HTTP précise sur une adresse précise (une route / endpoint):
Une requête GET sur /api/news

Une fonctionnalité du backend est déclenchée lorsque la bonne requête est faite sur la bonne adresse.
Cette fonctionnalité va donc dialoguer avec la BDD.

Et en réponse, nous allons renvoyer les données a l'utilisateur en format JSON:
{
    "title": "Mon super article",
    "content": "Le contenu de mon super article"
}

Si j'ai envie de créer une application permettant a des utilisateurs de faire des requêtes HTTP sur des adresses que j'aurais défini au préalable, puis recevoir des données en échange, j'aurai besoin de mettre en place différents composants :

1. Des ROUTES ! 
    /api/news
    /api/users
    /api/products

2. Des CONTROLLERS !
Les controllers sont le petit bout de code qui va REELLEMENT executer des ordres.
C'est un peu le chef d'orchestre, il reçoit une requête HTTP, il envoie des ordres a la BDD, et rerenvoie une réponse au client.

    NewsController
    UserController
    ProductController

    Get /api/news -> NewsController.getAllNews()

3. Des MODELS !
Les modèles, sont les fichiers qui vont définir la structure/schéma de données d'une TABLE.
C'est a dire, mon modèle news, pourras décrire par exemple qu'une News contient ceci :

    title, string
    content, text
    author, uuid
    publicationDate, date

Développement d'API :

1. Qu'est ce que c'est qu'une API/Webservices ?
    - Un backend indépendant avec lequel on peut communiquer via des requêtes HTTP(navigateur, extension vscode, postman, ou bien une app Front)

2. Qu'est ce que c'est que Node JS ? Express ! (npm)
    - NodeJS = Javascript coté serveur, Express = framework stable de NodeJS

3. Qu'est ce que c'est qu'une architecture MVC ?
    - Une structure de dossier/fichiers qui permet de séparer la logique en 3 composants principaux :
        - Model : Définition d'une structure de données
        - Controller : C'est ici ou la logique applicative se lance réellement, le chef d'orchestre
        - Route : C'est ici qu'on définis l'adresse ainsi que le type/la structure de requête a effectuer
